# 설치 및 사용 가이드

설치·환경 설정·실행·트러블슈팅을 한 문서에서 다룹니다.  
LLM은 `.env`의 `LLM_PROVIDER`로 선택 (claude | gemini | groq | ollama). 로컬 LLM은 [ollama.md](ollama.md) 참고.

---

## 1. 사전 요구사항

| 항목       | 요구 사항                                                                 |
| ---------- | ------------------------------------------------------------------------- |
| **Python** | 3.10 이상                                                                 |
| **pip**    | 최신 버전 권장                                                            |
| **LLM**    | Claude / Gemini / Groq API 키 또는 **Ollama**(로컬, API 키 불필요). `.env`의 `LLM_PROVIDER`에 맞춤 |

> **세 가지 방식으로 사용할 수 있습니다**
> - **① AI 코드 어시스턴트 방식** — Cursor 등 AI 도구에게 말로 시키면 RFP 분석부터 PPTX 생성까지 자동 처리
> - **② CLI(API) 방식** — `python main.py generate` 명령어로 실행. `.env`의 LLM_PROVIDER와 API 키 필요
> - **③ 수동 모드** — `python main.py generate <rfp> --manual` 후, `manual/N_step_{Phase명}_request.txt`의 프롬프트를 Gemini 등에 붙여넣고 응답을 `manual/N_step_{Phase명}_response.txt`에 저장한 뒤 `python main.py continue` 반복. API 비용·토큰 한도 없음

## 2. 설치

### 2-1. 프로젝트 클론

```bash
git clone https://github.com/your-username/proposal-agent.git
cd proposal-agent
```

### 2-2. 가상환경 생성 (권장)

```bash
python3 -m venv venv
source venv/bin/activate    # macOS/Linux
# venv\Scripts\activate     # Windows
```

### 2-3. 의존성 설치

```bash
pip install -r requirements.txt
```

설치되는 주요 패키지:

| 카테고리     | 패키지                                   | 용도             |
| ------------ | --------------------------------------- | ---------------- |
| AI           | `anthropic`, `google-genai`, `groq`     | LLM API 호출     |
| 문서 파싱    | `pypdf`, `pdfplumber`, `python-docx`     | PDF/DOCX 파싱    |
| PPTX 생성    | `python-pptx`                           | 파워포인트 생성   |

### 2-4. API 키 설정 (Claude / Gemini / Groq 사용 시)

`.env` 파일을 만듭니다. `cp .env.example .env` 후 값을 채웁니다.

```env
# 사용할 LLM 선택 (하나만): claude | gemini | groq | ollama
LLM_PROVIDER=gemini

# Claude 사용 시
# ANTHROPIC_API_KEY=...
# ANTHROPIC_MODEL=claude-3-5-sonnet-20241022

# Gemini 사용 시
# GEMINI_API_KEY=...
# GEMINI_MODEL=gemini-2.5-flash-lite

# Groq 사용 시 (무료 한도 넉넉)
# GROQ_API_KEY=...
# GROQ_MODEL=llama-3.1-8b-instant
```

### 2-5. Ollama 설치 (로컬 LLM 사용 시)

`LLM_PROVIDER=ollama` 로 사용하면 **API 키 없이** 로컬 LLM으로 동작합니다. 비용·요청 한도 제한이 없습니다.

#### Ollama 설치

| OS          | 방법                                                                                       |
| ----------- | ------------------------------------------------------------------------------------------ |
| **Windows** | [ollama.com/download](https://ollama.com/download)에서 OllamaSetup.exe 다운로드 후 실행. `ollama --version`으로 확인. |
| **macOS**   | [ollama.com/download](https://ollama.com/download)에서 macOS용 설치 → Applications로 설치 후 앱 실행. 메뉴바 Ollama 아이콘 확인. |
| **Linux**   | `curl -fsSL https://ollama.com/install.sh | sh` 실행. 이후 `ollama serve`로 서비스 기동. |

#### 모델 내려받기 및 확인

```bash
ollama pull llama3.2
ollama list   # 여기 나오는 이름을 .env 의 OLLAMA_MODEL 에 넣음
```

| 모델         | 명령                     | 비고             |
| ------------ | ------------------------ | ---------------- |
| llama3.2     | `ollama pull llama3.2`   | 기본 추천, 가벼움 |
| qwen2.5:7b   | `ollama pull qwen2.5:7b`  | 한글·다국어       |
| llama3.1:8b  | `ollama pull llama3.1:8b` | 품질·용량 균형    |

#### .env 설정 (Ollama 사용 시)

```env
LLM_PROVIDER=ollama
OLLAMA_BASE_URL=http://localhost:11434/v1
OLLAMA_MODEL=llama3.2:latest
```

상세 내용은 [3.ollama.md](3.ollama.md)를 참고하세요.

## 3. 사용법

### 방법 ① AI 코드 어시스턴트로 제안서 생성 (권장)

```
사용자: "제안요청서 폴더의 테스트 01 파일을 분석한 후 제안서를 제작해줘"

AI가 자동으로 수행:
  1. RFP Chunking으로 평가기준·요구사항 우선 파싱 (최대 40,000자)
  2. Impact-8 구조로 콘텐츠 기획 + Win Theme 3개 도출
  3. Cross-Phase Context + Industry Stats 주입으로 일관성 있는 제안서 생성
  4. PPTX 렌더링 (폰트 폴백 적용)
  5. Slide Quality Score 확인 → 품질 미달 슬라이드 수정
```

### 방법 ② CLI 명령어

#### (최초 1회) 회사 프로필 설정

```bash
python main.py setup-company
```

회사 정보를 대화형으로 입력하면 `company_data/company_profile.json`이 생성됩니다.
이후 모든 제안서의 **Phase 6(WHY US)**에 실제 역량·실적이 자동 반영됩니다.

#### 제안서 생성

```bash
python main.py generate input/rfp.pdf -n "프로젝트명" -c "발주처명"
```

| 옵션                   | 설명                     | 필수 |
| ---------------------- | ------------------------ | ---- |
| `input/rfp.pdf`        | RFP 파일 경로            | O    |
| `-n` / `--name`        | 프로젝트명               | O    |
| `-c` / `--client`      | 발주처명                 | O    |
| `-t` / `--type`       | 제안서 유형              | X (자동 판별) |
| `-d` / `--company`    | 회사 정보 JSON           | X (기본: company_data/company_profile.json) |
| `-o` / `--output`     | 출력 디렉토리            | X (기본: output/) |
| `-T` / `--template`   | PPTX 템플릿 파일명       | X (미지정 시 기본 디자인) |
| `--save-json`          | 콘텐츠 JSON 저장         | X    |
| `--manual` / `-m`     | 수동 모드 (LLM API 없이 파일 기반) | X    |

#### 수동 모드 (LLM API·과금 없이 진행)

API 호출 없이, 프롬프트를 사람이 Gemini(Google AI Studio) 등에 붙여넣고 응답을 파일로 저장한 뒤 단계별 진행합니다.

```bash
# 1. 수동 모드 시작 → manual/1_step_RFPAnalyzer_request.txt 생성
python main.py generate input/rfp.pdf --manual -n "프로젝트명" -c "발주처"

# 2. manual/1_step_RFPAnalyzer_request.txt 의 [시스템 프롬프트]·[사용자 메시지]를 https://aistudio.google.com/ 에 입력
# 3. Gemini 응답(JSON)을 manual/1_step_RFPAnalyzer_response.txt 에 붙여넣기
# 4. 다음 단계 진행 (Step 2~9 반복)
python main.py continue

# 진행 상태 확인
python main.py status
```

- **총 9단계**: Step 1 RFP 분석 → Step 2 Phase 0 (HOOK) → … → Step 9 Phase 7 (INVESTMENT) → PPTX 생성
- 작업 폴더 기본값: `manual/`. `--manual-dir` 로 변경 가능 (continue, status 동일)

#### 제안서 유형

| 유형 코드       | 설명              | Phase 4 비중 |
| --------------- | ----------------- | ------------ |
| `marketing_pr`  | 마케팅/PR/소셜미디어 | 40%          |
| `event`         | 이벤트/행사        | 45%          |
| `it_system`     | IT/시스템 구축     | 35%          |
| `public`        | 공공/입찰         | 30%          |
| `consulting`    | 컨설팅            | 30%          |
| `general`       | 일반 (기본값)      | 35%          |

#### RFP 분석만 수행

```bash
python main.py analyze input/rfp.pdf
```

#### 기타 명령어

```bash
python main.py continue   # 수동 모드: 현재 단계 응답 처리 후 다음 단계 (--manual 사용 시)
python main.py status     # 수동 모드 진행 상태 확인
python main.py types      # 제안서 유형 목록
python main.py templates  # PPTX 템플릿 목록
python main.py info       # Impact-8 프레임워크 설명
python main.py --help     # 전체 도움말
```

**지원 RFP 형식:** `.pdf`, `.docx`/`.doc`, `.txt`, `.pptx` (`.ppt` 구 형식 미지원)

## 6. 신기능 상세

### RFP Chunking (긴 RFP 완전 분석)
- 한국어 공공 문서 헤딩 패턴 인식으로 섹션 분류
- 평가기준·배점·요구사항 섹션을 HIGH 우선순위로 전문 포함
- 기존 25,000자 제한 → 최대 40,000자로 확장

### Slide Quality Scoring (자동 채점)
- Phase 완료 후 규칙 기반으로 각 슬라이드 자동 채점 (0~100점)
- 항목: Action Title 준수(25%), 내용 풍부성(30%), 구체성(25%), 플레이스홀더(15%), 유형 적합성(5%)
- 평균 60점 미만 Phase는 경고 로그 출력

### Cross-Phase Context (내러티브 일관성)
- 이전 Phase의 핵심 결론(Win Theme, KPI, 전략 방향)을 다음 Phase 프롬프트에 자동 주입
- Phase 간 메시지 단절 방지

### Industry Stats DB (수치 구체성)
- 업종별 검증 통계를 프롬프트에 자동 삽입
- 예: marketing_pr → SNS 활용률, 콘텐츠 마케팅 ROI 등
- `INDUSTRY_STATS_PATH` 환경변수로 커스텀 통계 JSON 지정 가능

### Phase Checkpoint (안정성)
- 각 Phase 완료 후 `output/_checkpoints/phase_N_*.json` 저장
- API 실패 시 성공한 Phase까지는 재실행 불필요

## 8. 트러블슈팅

### API 키 오류

```
ANTHROPIC_API_KEY / GEMINI_API_KEY / GROQ_API_KEY가 설정되지 않았습니다.
```

`.env`에서 `LLM_PROVIDER`와 선택한 프로바이더에 맞는 키 확인. **Ollama** 사용 시에는 API 키가 필요 없습니다.

### Ollama 관련

| 현상                            | 확인·조치                                                                 |
| ------------------------------- | ------------------------------------------------------------------------- |
| **Connection refused / 연결 실패** | Ollama 실행 여부 확인. Windows/macOS: 앱 실행, Linux: `ollama serve`      |
| **model not found**             | `OLLAMA_MODEL`을 `ollama list` 이름과 맞추기. `ollama pull <모델명>` 후 `.env` 설정 |
| **응답이 느림**                  | 모델 크기·PC 사양(GPU) 영향. 작은 모델(예: llama3.2) 또는 GPU 드라이버 확인 |
| **JSON 파싱 실패·형식 깨짐**     | `LLM_TEMPERATURE=0.3`으로 낮추거나, qwen2.5:7b·llama3.1:8b 등 큰 모델 사용   |

### PDF 파싱 오류

- 텍스트 기반 PDF인지 확인 (스캔 이미지 PDF 미지원)
- 암호화된 PDF 미지원

### PPTX 생성 오류

```bash
pip install --upgrade python-pptx
```

### 슬라이드 품질 경고 로그

```
WARNING | Phase 3 품질 점수: 45.2점 (기준: 60점 미만)
```

프롬프트를 강화하거나 `.env`에서 `MIN_QUALITY_SCORE=40` 으로 기준 완화 가능.

### 429 할당량 초과 (API 사용 시)

- `LLM_PROVIDER=groq` 로 변경(무료 한도 넉넉) 또는 `LLM_PROVIDER=ollama` 로 로컬 LLM 사용.
- `.env`에 `LLM_DELAY_SECONDS=10` 등으로 호출 간 대기 늘리기.

### Phase 생성 실패 후 재시작

- `ENABLE_CHECKPOINT=true`(기본값)이면 `output/_checkpoints/`에 성공한 Phase가 저장됨. 실패한 Phase부터 수동 재실행 가능.

### 폰트 표시 문제

- 템플릿에 Pretendard 등 미설치 폰트가 있으면 **맑은 고딕으로 자동 폴백**됨. 해당 폰트를 시스템에 설치하면 자동 적용.
