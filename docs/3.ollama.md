# Ollama 로컬 LLM 연동 가이드

제안서 에이전트(proposal-agent)를 **API 비용·무료 티어 한도 없이** 쓰려면 로컬에 **Ollama**를 설치한 뒤 LLM을 Ollama로 연동하면 됩니다.  
이 문서는 **Ollama 설치**, **모델 받기**, **에이전트 설정·실행**, **문제 해결**만 정리합니다. (소스 수정은 이미 반영되어 있음.)

---

## 1. 왜 Ollama를 쓰나요?

| 항목          | 설명                                                                                                                                 |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| **API 키**    | 불필요. 로컬 `http://localhost:11434` 로만 통신합니다.                                                                               |
| **비용·한도** | 로컬 실행이라 **비용 없음**, **요청 한도 없음**.                                                                                     |
| **품질·속도** | 사용하는 모델(예: llama3.2, qwen2.5)과 PC 사양(CPU/GPU, RAM)에 따라 달라집니다. GPU가 있으면 더 빠르고 품질이 좋습니다.             |

---

## 2. Ollama 설치

### 2.1 Windows

1. **[ollama.com/download](https://ollama.com/download)** 에서 **OllamaSetup.exe** 다운로드.
2. **OllamaSetup.exe** 실행 → UAC 승인 → 설치 완료. (사용자 홈 기준 설치, 관리자 권한 불필요.)
3. 설치 후 Ollama가 백그라운드에서 실행되며 **http://localhost:11434** 에 API 서버가 떠 있습니다.
4. 터미널에서 확인:
   ```powershell
   ollama --version
   ```

### 2.2 macOS

1. [ollama.com/download](https://ollama.com/download) 에서 macOS용 설치 파일 다운로드.
2. 다운로드한 이미지를 **Applications** 로 드래그하거나 설치 절차 진행.
3. 앱 실행 후 **메뉴바에 Ollama 아이콘**이 보이면 서버가 동작 중입니다.

### 2.3 Linux

1. 설치:
   ```bash
   curl -fsSL https://ollama.com/install.sh | sh
   ```
2. 서비스 기동 (일부 배포판은 설치 시 자동 등록):
   ```bash
   ollama serve
   ```

## 3. 모델 내려받기 및 확인

### 3.1 모델 받기

```bash
ollama pull llama3.2
```

- 다른 모델 예시:
  - `ollama pull qwen2.5:7b` (다국어·한글에 유리)
  - `ollama pull llama3.1:8b`

### 3.2 이미 받은 모델 목록

```bash
ollama list
```

- 여기 나오는 **이름**을 `.env` 의 `OLLAMA_MODEL` 에 그대로 넣으면 됩니다.

### 3.3 제안서 생성에 쓸 만한 모델 예시

| 모델              | 명령                        | 비고             |
| ----------------- | --------------------------- | ---------------- |
| **llama3.2**      | `ollama pull llama3.2`      | 기본 추천, 가벼움 |
| **qwen2.5:7b**    | `ollama pull qwen2.5:7b`    | 한글·다국어       |
| **llama3.1:8b**   | `ollama pull llama3.1:8b`    | 품질·용량 균형    |

## 4. 에이전트에서 Ollama 사용하기

### 4.1 환경 변수 (.env)

```env
LLM_PROVIDER=ollama
OLLAMA_BASE_URL=http://localhost:11434/v1
OLLAMA_MODEL=llama3.2:latest
```

- **OLLAMA_BASE_URL**: 기본값이 `http://localhost:11434/v1` 이므로 생략 가능.
- **OLLAMA_MODEL**: `ollama list` 에 나오는 이름과 맞추기 (예: `llama3.2:latest`, `qwen2.5:7b`).

### 4.2 실행 전 확인

1. **Ollama가 켜져 있는지**
   - Windows/macOS: Ollama 앱이 실행 중인지 확인.
   - Linux: `ollama serve` 또는 해당 서비스가 떠 있는지 확인.
2. **API 동작 확인** (선택):
   ```bash
   curl http://localhost:11434/api/tags
   ```
   - 모델 목록이 JSON으로 나오면 정상입니다.

### 4.3 제안서 생성·분석 실행

```bash
# 가상환경 활성화 후
pip install -r requirements.txt

# 제안서 생성 (RFP 경로는 실제 파일로 변경)
python main.py generate input/sample.txt

# RFP 분석만
python main.py analyze input/sample.txt
```

- 콘솔에 **Ollama (로컬)** 로 표시되면 Ollama 연동으로 동작하는 것입니다.

## 5. 문제 해결

| 현상                           | 확인·조치                                                                                       |
| ------------------------------ | ---------------------------------------------------------------------------------------------- |
| **Connection refused / 연결 실패** | Ollama가 꺼져 있음. Windows/macOS는 앱 실행, Linux는 `ollama serve` 또는 서비스 시작.          |
| **model not found**            | `OLLAMA_MODEL` 이 `ollama list` 에 있는 이름과 다름. `ollama pull <모델명>` 후 같은 이름으로 설정. |
| **응답이 느림**                 | 모델 크기·PC 사양(특히 GPU) 영향. 더 작은 모델로 시험하거나, GPU 드라이버·CUDA 확인.             |
| **JSON 파싱 실패·형식 깨짐**    | `LLM_TEMPERATURE=0.3` 으로 낮추거나, 더 큰 모델(예: qwen2.5:7b, llama3.1:8b) 사용.              |

---

## 6. 요약

1. **Ollama 설치**: [ollama.com/download](https://ollama.com/download) 에서 OS에 맞게 설치.
2. **모델 받기**: `ollama pull llama3.2` (또는 원하는 모델).
3. **에이전트 설정**: `.env` 에 `LLM_PROVIDER=ollama`, `OLLAMA_MODEL=llama3.2:latest` (또는 사용할 모델명) 설정.
4. **실행**: `python main.py generate input/...` 또는 `python main.py analyze input/...`.

이렇게 하면 API 비용과 클라우드 무료 티어 한도 없이 로컬 LLM으로 제안서 에이전트를 사용할 수 있습니다.
