# proposal-agent 실행 가이드 (Claude / Gemini / Groq)

프로젝트 설치·설정 후 RFP를 분석하고 제안서 PPTX를 생성하는 방법입니다.  
**사용할 LLM은 .env의 `LLM_PROVIDER`로 선택합니다 (claude | gemini | groq).**

---

## 1. 사전 준비

### 1-1. LLM API 키 발급 (선택한 프로바이더별 1개)

| LLM_PROVIDER | 발급처 | 비고 |
|--------------|--------|------|
| **claude** | [Anthropic Console](https://console.anthropic.com) | Claude (Anthropic) API |
| **gemini** | [Google AI Studio](https://aistudio.google.com/apikey) | 무료 한도 있음, 429 시 대기 또는 다른 프로바이더로 전환 |
| **groq** | [Groq Console](https://console.groq.com) | 무료 한도 넉넉, 429 대응에 유리 |

1. 위 표에서 사용할 프로바이더 하나를 골라 해당 사이트 접속
2. 회원가입/로그인 후 **API Key** 생성
3. 생성된 키를 복사해 두기 (한 번만 표시될 수 있음)

### 1-2. Python 환경

- **Python 3.10 이상** 필요
- 가상환경 사용 권장

### 2-4. API 키 및 LLM 설정

프로젝트 **루트**에 `.env` 파일을 만듭니다. `cp .env.example .env` 후 값을 채우거나, 직접 아래 내용을 참고해 작성합니다.

**공통: 사용할 LLM 선택**

```env
# 사용할 LLM: claude | gemini | groq (하나만 선택)
LLM_PROVIDER=gemini
```

**옵션 A — Claude (Anthropic)**  
`LLM_PROVIDER=claude` 로 두고 아래 변수 설정:

```env
LLM_PROVIDER=claude
ANTHROPIC_API_KEY=여기에_발급받은_키_입력
ANTHROPIC_MODEL=claude-3-5-sonnet-20241022
```

**옵션 B — Gemini**  
`LLM_PROVIDER=gemini` 로 두고 아래 변수 설정:

```env
LLM_PROVIDER=gemini
GEMINI_API_KEY=여기에_발급받은_키_입력
GEMINI_MODEL=gemini-2.5-flash-lite
# 429 나올 때 호출 간 대기(초)
LLM_DELAY_SECONDS=8
```

**옵션 C — Groq (무료 한도 넉넉)**  
`LLM_PROVIDER=groq` 로 두고 아래 변수 설정:

```env
LLM_PROVIDER=groq
GROQ_API_KEY=여기에_발급받은_키_입력
GROQ_MODEL=llama-3.1-8b-instant
```

| 프로바이더 | 필수 환경 변수 | 모델 예시 |
|------------|----------------|-----------|
| claude | `ANTHROPIC_API_KEY` | `claude-3-5-sonnet-20241022` |
| gemini | `GEMINI_API_KEY` | `gemini-2.5-flash-lite` |
| groq | `GROQ_API_KEY` | `llama-3.1-8b-instant` |

**선택 환경 변수** (`.env.example` 참고): `LOG_LEVEL`, `LLM_MAX_TOKENS`, `LLM_RETRY_COUNT`, `LLM_RETRY_BASE_DELAY`, `LLM_DELAY_SECONDS`(호출 간 대기), `GROQ_MAX_USER_MESSAGE_CHARS`

> - `.env`는 `.gitignore`에 포함되어 있어 Git에 커밋되지 않습니다. API 키는 저장소에 올리지 마세요.  
> - 429(할당량 초과)가 나오면 `LLM_PROVIDER=groq` 로 바꾸고 `GROQ_API_KEY`만 설정해도 동작합니다.

### 2-5. input / output 폴더

- 프로젝트에 **`input/`**, **`output/`** 폴더가 이미 있습니다.
- **RFP 파일**을 `input/`에 넣고, 생성된 제안서는 **`output/`**에 저장됩니다.

**지원 RFP 형식:**

| 확장자 | 형식 | 비고 |
|--------|------|------|
| `.pdf` | PDF | |
| `.docx`, `.doc` | Word 문서 | |
| `.txt` | 텍스트 파일 | UTF-8/CP949 등 자동 시도 |
| `.pptx` | PowerPoint | 슬라이드 텍스트·테이블 추출 |

**templates 폴더:** PPTX 템플릿(`templates/*.pptx`)을 두면 해당 파일을 기반으로 제안서가 생성됩니다. `templates/` 폴더는 `.gitignore`에 포함되어 있어 로컬에서만 사용할 수 있습니다.

## 3. 실행 방법

### 3-1. RFP 분석만 하기 (제안서 생성 없음)

```bash
python main.py analyze input/제안요청서.pdf
```

- 파일명·확장자만 바꿔서 사용 가능 (`.pdf`, `.docx`, `.txt`, `.pptx`)
- 분석 결과가 터미널에 출력됩니다.

**예: input 폴더에 sample.txt가 있는 경우**
```bash
python main.py analyze input/sample.txt
```

### 3-2. 제안서 생성 (PPTX 생성)

```bash
python main.py generate input/제안요청서.pdf -n "프로젝트명" -c "발주처명"
```

**옵션:**

| 옵션 | 설명 | 예시 |
|------|------|------|
| `-n` / `--name` | 프로젝트명 | `-n "2026년 SNS 운영"` |
| `-c` / `--client` | 발주처명 | `-c "[발주처명]"` |
| `-t` / `--type` | 제안서 유형 | `-t marketing_pr` |
| `-d` / `--company` | 회사 정보 JSON 경로 (기본: company_data/company_profile.json) | `-d company_data/company_profile.json` |
| `-o` / `--output` | 출력 폴더 (기본: output) | `-o output` |
| `-T` / `--template` | PPTX 템플릿 파일명(확장자 제외). 미지정 시 템플릿 없이 기본 디자인으로 생성(권장) | `-T guide_template` |
| `--save-json` | 콘텐츠 JSON 저장 | 플래그만 추가 |

**제안서 유형 (`-t`):**

- `marketing_pr` — 마케팅/PR/소셜미디어
- `event` — 이벤트/행사
- `it_system` — IT/시스템
- `public` — 공공/입찰
- `consulting` — 컨설팅
- `general` — 일반 (기본, 미지정 시 자동 판별)

**예시 (TXT RFP, 마케팅/PR):**
```bash
python main.py generate input/sample.txt -n "2026년 SNS 채널 운영" -c "[발주처명]" -t marketing_pr
```

생성된 PPTX는 **`output/프로젝트명_YYYYMMDDHHmmssfff.pptx`** 형식으로 저장됩니다 (동일 프로젝트명 재실행 시 덮어쓰기 방지).

### 3-3. 기타 명령어

```bash
# 사용 가능한 제안서 유형 목록
python main.py types

# 사용 가능한 PPTX 템플릿 목록 (guide_template 기본, templates/*.pptx)

# 전체 도움말

```

## 4. 실행 흐름 (generate 시)

1. **Phase 1: 콘텐츠 생성 (설정된 LLM)**  
   RFP 파싱(PDF/DOCX/TXT/PPTX) → LLM 분석 → Phase 0~7 콘텐츠 생성  
   (LLM은 .env의 `LLM_PROVIDER`에 따라 Claude / Gemini / Groq 중 하나 사용)
2. **Phase 2: PPTX 생성**  
   생성된 콘텐츠를 Modern 스타일 PPTX로 변환  
3. **완료**  
   `output/프로젝트명_YYYYMMDDHHmmssfff.pptx` 생성

처음 실행 시 LLM API 호출이 여러 번 이루어져 **1~3분** 정도 걸릴 수 있습니다.

## 5. 문제 해결

### "ANTHROPIC_API_KEY / GEMINI_API_KEY / GROQ_API_KEY가 설정되지 않았습니다"

- 프로젝트 **루트**에 `.env` 파일이 있는지 확인
- `.env` 안에 `LLM_PROVIDER=claude|gemini|groq` 와 **선택한 프로바이더에 맞는** API 키가 있는지 확인:
  - `LLM_PROVIDER=claude` → `ANTHROPIC_API_KEY=...`
  - `LLM_PROVIDER=gemini` → `GEMINI_API_KEY=...`
  - `LLM_PROVIDER=groq` → `GROQ_API_KEY=...`
- 키는 따옴표 없이 한 줄로 입력 가능

### "지원하지 않는 파일 형식"

- `generate` / `analyze`에 넘기는 파일이 **PDF**, **DOCX**, **TXT**, **PPTX** 중 하나인지 확인
- 구 PowerPoint 형식 **`.ppt`** 는 지원하지 않습니다. **`.pptx`** 만 가능합니다.

### input 폴더에 RFP가 없음

- RFP 파일(PDF/DOCX/TXT/PPTX)을 **`input/`** 폴더에 넣은 뒤 실행  
  예: `python main.py generate input/파일명.txt -n "프로젝트" -c "발주처"`

### 429 할당량 초과 (Gemini 사용 시)

- **원인:** Google AI Studio 무료 한도 초과.
- **해결:**
  1. **다른 LLM 사용**  
     `.env`에서 `LLM_PROVIDER=groq` 로 바꾸고 `GROQ_API_KEY` 설정 (무료 한도 넉넉).
  2. **Claude 사용**  
     `LLM_PROVIDER=claude`, `ANTHROPIC_API_KEY` 설정.
  3. **Gemini만 쓸 때**  
     `.env` 공통 항목에서 `LLM_DELAY_SECONDS=8` 등으로 호출 간 대기 늘리기, 또는 모델을 `gemini-2.0-flash-lite` 등으로 변경.

### 회사 정보 반영

- **`company_data/company_profile.json`** 에 회사 정보를 두면 제안서에 반영됩니다.
- `generate` 시 `-d company_data/company_profile.json` 처럼 `--company` 옵션으로 경로 지정 가능 (기본값이 이 경로입니다).
