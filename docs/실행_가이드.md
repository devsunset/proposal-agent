# proposal-agent 실행 가이드 (Claude / Gemini / Groq)

프로젝트 설치·설정 후 RFP를 분석하고 제안서 PPTX를 생성하는 방법입니다.
**사용할 LLM은 .env의 `LLM_PROVIDER`로 선택합니다 (claude | gemini | groq).**

---

## 1. 사전 준비

### 1-1. LLM API 키 발급 (선택한 프로바이더별 1개)

| LLM_PROVIDER | 발급처 | 비고 |
|--------------|--------|------|
| **claude** | [Anthropic Console](https://console.anthropic.com) | Claude API |
| **gemini** | [Google AI Studio](https://aistudio.google.com/apikey) | 무료 한도 있음, 429 시 대기 또는 다른 프로바이더로 전환 |
| **groq** | [Groq Console](https://console.groq.com) | 무료 한도 넉넉, 429 대응에 유리 |

1. 위 표에서 사용할 프로바이더 하나를 골라 해당 사이트 접속
2. 회원가입/로그인 후 **API Key** 생성
3. 생성된 키를 복사해 두기

### 1-2. Python 환경

- **Python 3.10 이상** 필요
- 가상환경 사용 권장


### 2-4. API 키 및 LLM 설정

프로젝트 **루트**에 `.env` 파일을 만듭니다. `cp .env.example .env` 후 값을 채웁니다.

**공통: 사용할 LLM 선택**

```env
LLM_PROVIDER=gemini
```

**옵션 A — Claude:**

```env
LLM_PROVIDER=claude
ANTHROPIC_API_KEY=여기에_발급받은_키_입력
ANTHROPIC_MODEL=claude-3-5-sonnet-20241022
```

**옵션 B — Gemini:**

```env
LLM_PROVIDER=gemini
GEMINI_API_KEY=여기에_발급받은_키_입력
GEMINI_MODEL=gemini-2.5-flash-lite
LLM_DELAY_SECONDS=8
```

**옵션 C — Groq (무료 한도 넉넉):**

```env
LLM_PROVIDER=groq
GROQ_API_KEY=여기에_발급받은_키_입력
GROQ_MODEL=llama-3.1-8b-instant
```

| 프로바이더 | 필수 환경 변수 | 모델 예시 |
|------------|----------------|-----------|
| claude | `ANTHROPIC_API_KEY` | `claude-3-5-sonnet-20241022` |
| gemini | `GEMINI_API_KEY` | `gemini-2.5-flash-lite` |
| groq | `GROQ_API_KEY` | `llama-3.1-8b-instant` |

> `.env`는 `.gitignore`에 포함되어 있어 Git에 커밋되지 않습니다.

### 2-5. 고도화 옵션 (.env.example 참고)

기본값으로도 잘 동작합니다. 필요 시만 수정하세요.

| 환경 변수 | 기본값 | 설명 |
|-----------|--------|------|
| `ENABLE_RFP_CHUNKING` | `true` | 평가기준·요구사항 섹션 우선 포함 청킹 |
| `RFP_CHUNK_MAX_CHARS` | `40000` | 청킹 후 최대 문자 수 |
| `ENABLE_QUALITY_SCORING` | `true` | 슬라이드 자동 채점 (Action Title·구체성 등) |
| `MIN_QUALITY_SCORE` | `60` | Phase 평균 품질 기준 점수 |
| `ENABLE_CROSS_PHASE_CONTEXT` | `true` | Phase 간 내러티브 일관성 |
| `ENABLE_CHECKPOINT` | `true` | Phase별 JSON 체크포인트 저장 |
| `ENABLE_INDUSTRY_STATS` | `true` | 업종별 통계 자동 주입 |
| `INDUSTRY_STATS_PATH` | (내장) | 커스텀 통계 JSON 파일 경로 |
| `PROMPT_VERSION` | (없음) | 프롬프트 버전 디렉토리 |
| `ENABLE_SELF_REFINEMENT` | `false` | Draft→Critique→Refine (토큰 3배, 실험적) |

### 2-6. input / output 폴더

- RFP 파일을 `input/`에 넣고, 생성된 제안서는 `output/`에 저장됩니다.
- `output/_checkpoints/` — Phase 체크포인트 JSON 자동 저장 (API 실패 시 복구용)

**지원 RFP 형식:**

| 확장자 | 형식 |
|--------|------|
| `.pdf` | PDF |
| `.docx`, `.doc` | Word 문서 |
| `.txt` | 텍스트 파일 |
| `.pptx` | PowerPoint |

## 3. 실행 방법

### 3-0. (최초 1회) 회사 프로필 설정

```bash
python main.py setup-company
```

회사명·주요 서비스·수행 실적·핵심 인력 등을 입력하면 `company_data/company_profile.json`에 저장됩니다.
이후 제안서 생성 시 **Phase 6(WHY US)**에 실제 역량·실적이 자동으로 반영됩니다.

### 3-1. RFP 분석만 하기

```bash
python main.py analyze input/제안요청서.pdf
```

분석 결과(프로젝트명, 발주처, 요구사항, 평가기준, Pain Point, Win Theme 후보 등)가 터미널에 출력됩니다.

### 3-2. 제안서 생성 (PPTX 생성)

```bash
python main.py generate input/제안요청서.pdf -n "프로젝트명" -c "발주처명"
```

**옵션:**

| 옵션 | 설명 | 예시 |
|------|------|------|
| `-n` / `--name` | 프로젝트명 | `-n "2026년 SNS 운영"` |
| `-c` / `--client` | 발주처명 | `-c "서울시"` |
| `-t` / `--type` | 제안서 유형 | `-t marketing_pr` |
| `-d` / `--company` | 회사 정보 JSON 경로 | `-d company_data/company_profile.json` |
| `-o` / `--output` | 출력 폴더 (기본: output) | `-o output` |
| `-T` / `--template` | PPTX 템플릿 파일명(확장자 제외) | `-T guide_template` |
| `--save-json` | 콘텐츠 JSON 저장 | 플래그만 추가 |

**제안서 유형 (`-t`):**

- `marketing_pr` — 마케팅/PR/소셜미디어
- `event` — 이벤트/행사
- `it_system` — IT/시스템
- `public` — 공공/입찰
- `consulting` — 컨설팅
- `general` — 일반 (미지정 시 자동 판별)

**예시:**
```bash
python main.py generate input/sample.txt -n "2026년 SNS 채널 운영" -c "서울시" -t marketing_pr
```

생성된 PPTX: `output/프로젝트명_YYYYMMDDHHmmssfff.pptx`

## 4. 실행 흐름 (generate 시)

```
1. 문서 파싱 (PDF/DOCX/TXT/PPTX)
2. RFP 분석 (LLM)
   └─ RFP Chunking: 섹션 우선순위 기반, 평가기준·요구사항 우선 포함 (최대 40,000자)
3. 콘텐츠 생성 (Phase 0~7)
   ├─ Cross-Phase Context: 이전 Phase 결론 자동 전달
   ├─ Industry Stats DB: 업종별 통계 주입
   ├─ Negative Prompts: 플레이스홀더·막연한 표현 금지
   ├─ Slide Quality Scoring: 각 Phase 완료 후 자동 채점 (경고 로그)
   └─ Phase Checkpoint: output/_checkpoints/ 자동 저장
4. PPTX 생성 (Modern 스타일, 폰트 폴백 적용)
5. 완료: output/프로젝트명_YYYYMMDDHHmmssfff.pptx
```

처음 실행 시 LLM API 호출이 여러 번 이루어져 **1~3분** 정도 걸릴 수 있습니다.

## 5. 문제 해결

### API 키 오류

```
ANTHROPIC_API_KEY / GEMINI_API_KEY / GROQ_API_KEY가 설정되지 않았습니다
```

- `.env`에 `LLM_PROVIDER=claude|gemini|groq`와 해당 API 키 확인
  - `LLM_PROVIDER=claude` → `ANTHROPIC_API_KEY=...`
  - `LLM_PROVIDER=gemini` → `GEMINI_API_KEY=...`
  - `LLM_PROVIDER=groq` → `GROQ_API_KEY=...`

### 지원하지 않는 파일 형식

- PDF, DOCX, TXT, PPTX만 지원합니다. `.ppt` (구 형식)는 미지원.

### 429 할당량 초과 (Gemini 사용 시)

1. `LLM_PROVIDER=groq` 로 변경 (무료 한도 넉넉)
2. `LLM_DELAY_SECONDS=10` 등으로 호출 간 대기 늘리기

### Phase 생성 실패 후 재시작

- `ENABLE_CHECKPOINT=true`(기본값)이면 `output/_checkpoints/`에 성공한 Phase가 저장됩니다.
- 실패 Phase부터 재실행 가능 (수동 복구는 향후 자동화 예정).

### 폰트 표시 문제

- 템플릿 PPTX에 Pretendard 등 미설치 폰트가 있으면 **맑은 고딕으로 자동 폴백**됩니다.
- 수동 설정: `.env`에 특정 폰트 지정 없이 시스템에 해당 폰트를 설치하면 자동 적용됩니다.
